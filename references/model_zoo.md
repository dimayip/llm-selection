## 五、选型关键维度详解

### 5.1 能力维度

| 维度 | 说明 | 评估方法 |
|------|------|---------|
| 推理能力 | 逻辑推理、数学计算、复杂问题分析 | MMLU、GPQA、MATH 基准 |
| 代码能力 | 代码生成、调试、代码解释 | HumanEval、MBPP 基准 |
| 知识广度 | 训练数据覆盖范围、知识截止日期 | 事实性问答测试 |
| 多语言能力 | 特别是中文支持程度 | CMMLU、C-Eval 中文基准 |
| 多模态 | 图像理解、音频处理、视频理解 | MMMU、Video-MME 基准 |

### 5.2 工程维度

| 维度 | 说明 | 关注点 |
|------|------|--------|
| 上下文长度 | 能处理的多长文本 | 长文档场景必须 ≥ 需求 |
| 推理速度 | Token 生成速度（tokens/s） | 实时对话场景关注首 token 延迟 |
| 并发能力 | 同时处理多少请求 | 高并发场景关注 RPM/TPM 限制 |
| 稳定性 | API 可用性 SLA | 生产环境关注 99.9% 以上 SLA |
| 合规 | 数据隐私、内容审核政策 | 国内业务必须数据不离境 |

### 5.3 成本维度

| 计费模式 | 说明 | 适用场景 |
|---------|------|---------|
| 按 token 计费 | 输入/输出分别计费 | 大部分 API 调用 |
| 包月订阅 | 固定费用，不限量或限流量 | 高频调用场景 |
| 自部署 | GPU 租赁费 + 电费 + 运维 | 数据隐私要求高场景 |
| 免费额度 | 各平台提供一定免费 quota | 试用、低频调用 |

### 5.4 访问维度

| 问题 | 国际模型 | 国内模型 |
|------|---------|---------|
| 是否需要科学上网 | ✅ 是 | ❌ 否 |
| API 申请难度 | 高（部分需 waitlist） | 低（大部分开放注册） |
| 速率限制 | 较严（RPM/TPM 限制） | 相对宽松 |
| 数据合规 | 数据出境风险 | 数据留在国内 |

## 七、快速选型决策树

```
开始
 ├─ 数据必须留在国内？
 │   ├─ 是 → 国内模型（混元/通义/DeepSeek/GLM/豆包）
 │   └─ 否 → 国际模型（GPT/Claude/Gemini）
 │
 ├─ 需要本地部署？
 │   ├─ 是 → 开源模型（Qwen 2.5 / DeepSeek R1 蒸馏版 / GLM-4）
 │   └─ 否 → 继续
 │
 ├─ 主要场景？
 │   ├─ 代码 → Claude 4 / GPT-4o / Qwen Coder
 │   ├─ 推理 → GPT-o3 / DeepSeek R1 / 混元 T1
 │   ├─ 长文档 → Gemini 2.5 Pro / Qwen 2.5 / Llama 4
 │   ├─ 多模态 → GPT-4o / Gemini 2.5 Pro / 通义 VL
 │   ├─ 中文创作 → 混元 / 通义 / 文心
 │   └─ 实时对话 → GPT-4o mini / 豆包 / 通义 Plus
 │
 ├─ 预算？
 │   ├─ 充足 → 旗舰模型（GPT-4o / Claude 4 / 混元 Turbo）
 │   ├─ 有限 → 性价比模型（DeepSeek V3 / 豆包 / 通义 Plus）
 │   └─ 极低 → 开源本地部署（Qwen 2.5 14B/32B）
 │
 └─ 输出 Top 3 推荐模型，附成本和落地建议
```
